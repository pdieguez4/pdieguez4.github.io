QUÈ HEM FET?

El nostre projecte es basa en la creació d’un sistema interactiu capaç d’escoltar i respondre amb so, una mena de dispositiu que “escolta” l’entorn a través d’un micròfon i reprodueix sons o melodies mitjançant un altaveu, controlat amb una placa microcontroladora.

Inicialment, vam crear el nostre codi font per generar notes musicals amb Arduino i reproduir-les amb un altaveu. La primera placa que vam utilitzar va ser la ESP32-S3 WROOM, que va ser també la primera a donar-nos problemes tècnics. Aquests són alguns dels errors amb què ens vam trobar:

Que significa l'error GND a Arduino?

No és un error oficial del programari d'Arduino, sinó un problema habitual de maquinari.

Pot presentar-se de diverses formes:

1. Falta de connexió al GND

Si connectes sensors o mòduls sense unir els seus GND amb el de l’Arduino, no hi ha una referència comuna de voltatge. Resultat: fallades aleatòries o mal funcionament.

2. Connexions dolentes o cables solts

Un GND mal connectat pot provocar errors de lectura, reinicis o fallades de comunicació.

3. GNDs no compartits entre diferents fonts

Si utilitzes, per exemple, una bateria i un USB sense unir els seus GNDs, els dispositius no “parlen el mateix idioma elèctric”.

Què significa l'error "Exit Status"?

L'exit status (o codi de sortida) és un número que un programa retorna al sistema operatiu quan acaba la seva execució.

En el cas de l'IDE d'Arduino, aquest missatge apareix quan alguna cosa falla durant la compilació o la càrrega del programa a la placa.

Per exemple:

  • exit status 0: Tot ha anat bé, sense errors.
  • exit status 1: Hi ha hagut un error de compilació, com ara una ordre mal escrita, una llibreria que falta o un problema de sintaxi.
  • exit status 255: Sol indicar un error greu, com que el port USB no està disponible.

Aquest número per si sol no dona gaire informació; el més important és llegir el missatge d'error complet que apareix just abans a la consola de l'IDE. Allà trobaràs la causa real del problema.

Aquest es el codi que hem ulitzat:

const int micPin = A0;
const int speakerPin = 5;

const int threshold = 300;

#define DO  262
#define RE  294
#define MI  330
#define FA  349
#define SOL 392
#define LA  440
#define SI  494
#define DO2 523

const int baseDuration = 400;

int melody[] = {
  DO, DO, RE, MI,
  MI, RE, DO, RE,
  MI, MI, MI, 0,
  RE, RE, RE, 0,
  DO, DO, DO, 0,
  DO, RE, MI, MI, RE, DO
};

int noteDurations[] = {
  400, 400, 400, 400,
  400, 400, 400, 400,
  600, 400, 400, 400,
  600, 400, 400, 400,
  600, 400, 400, 400,
  400, 400, 400, 400, 400, 600
};

void setup() {
  pinMode(speakerPin, OUTPUT);
  Serial.begin(115200);
}

void loop() {
  int micValue = analogRead(micPin);
  Serial.println(micValue);

  if (micValue > threshold) {
    for (int i = 0; i < sizeof(melody) / sizeof(melody[0]); i++) {
      int note = melody[i];
      int duration = noteDurations[i];

      if (note == 0) {
        noTone(speakerPin);
      } else {
        tone(speakerPin, note, duration);
      }
      delay(duration * 1.3);
    }
    delay(1000);
  } else {
    noTone(speakerPin);
  }

  delay(50);
}
